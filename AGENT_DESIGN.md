### 2.3 关键发现

| 发现 | 说明 |
|------|------|
| **MCP 是主流集成模式** | Nexus、obsidian_mcp、ClawMem 等都通过 MCP 连接 AI ↔ 知识库 |
| **混合检索成为标配** | BM25 + 向量搜索 + RRF 融合是 2025-2026 的标准架构 |
| **本地优先是趋势** | 所有新项目都本地优先，不用云 |
| **Claude Code hooks 驱动记忆** | 通过 SessionStart/UserPromptSubmit/Stop 等 hook 自动捕获 |
| **知识生命周期管理** | 记忆会过期、会矛盾检测、会自动整理——不是"存了就完事" |

## 三、我们自己的方案设计

### 3.1 为什么不自建全套

看了这么多项目后，一个重要的决定：

> **我们不自建笔记编辑器、不自建渲染引擎、不自建图谱可视化。**
> 
> Obsidian 在这些方面已经做得很好了，我们用它的 `.md` 格式 + 目录结构。
> 我们要做的是 **Agent 大脑层**——理解、链接、整理、检索。

### 3.3 核心流程：一次典型交互

```
用户: "记一下：我决定用 FastAPI 重写后端，因为 Django 太重了"

                    ↓
Agent 理解意图 → "这是决策记录 / 笔记创建"

                    ↓
1. 搜索知识库是否有相关笔记（"FastAPI" "Django" "重写"）
2. 如果找到 → 读取已有笔记，了解上下文
3. 创建新笔记 / 更新已有笔记

                    ↓
输出笔记草稿:
---
title: 决定用 FastAPI 重写后端
type: decision
date: 2025-06-25
tags: [backend, FastAPI, Django, architecture]
links: [[后端架构讨论]]
---

**决策**: 用 FastAPI 重写后端
**原因**: Django 太重，FastAPI 更适合当前需求
**上下文**: 参考 [[后端架构讨论]]

---

Agent: "已记录。我发现你之前有一篇 [[后端架构讨论]] 提到了相关考虑，已自动建立链接。要不要我顺便总结一下两者的对比？"
```

### 3.4 与对标产品的差异定位

```
                    Cloud / 重管理
                         │
                         │ Notion AI
                         │
      NotebookLM ────────┼────────  Mem
      深研究              │          零摩擦
                         │
                  obsidian-mcp ◄───  我们在这里
                  Nexus MCP    │
                         │
                         │ Obsidian + 插件
                         ▼
                    Local / 重思考
```

**我们的独特价值：**

| 对比产品 | 他们没有的 | 我们有的 |
|---------|-----------|---------|
| **Notion AI** | 本地存储、知识图谱 | 全本地 + 双向链接 + Claude 质量 |
| **NotebookLM** | 记录能力、可搜索的持久笔记 | 长期积累 + 自动整理 + 可修改 |
| **Obsidian 插件** | 原生 AI 理解、自动链接 | Agent 主动思考 + 自动建立联系 |
| **Mem** | 结构化输出、日程管理 | 既捕捉也整理 + 日程 + Claude |

### 3.5 文件格式设计

每篇笔记是一个标准 Markdown 文件，兼容 Obsidian：

```markdown
---
title: 用 FastAPI 重写后端
type: decision | note | idea | daily | reference
date: 2025-06-25
tags: [backend, FastAPI]
links: [[后端架构讨论]], [[技术选型原则]]
status: active | archived | draft
---

## 背景
[正文内容]

## 结论
...

## 相关
- [[后端架构讨论]]
- [[技术选型原则]]
```

## 五、总结

| 问题 | 答案 |
|------|------|
| **做什么** | 一个基于 Claude 的个人知识管理 Agent |
| **不做什么** | 不做笔记编辑器/渲染器——Obsidian 负责 UI |
| **做什么** | 理解、链接、整理、检索——Agent 大脑 |
| **对标谁** | 填补 Obsidian 插件和 NotebookLM 之间的空白 |
| **独特价值** | 本地优先 + 主动链接 + Claude 深度理解 |
| **第一周目标** | 能说一句话就记下一张格式化笔记，自动关联已有知识 |
