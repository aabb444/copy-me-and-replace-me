# 长城 AI 科技战略 — 个人知识管理 Agent 架构设计

> 核心定位：一个围绕 **长城 AI 科技战略** 领域的个人知识管理 Agent，
> 集任务/日程管理、智能笔记分类存储、按标签汇总回顾于一体。

## 二、核心流程详解

### 2.1 任务创建流程

```
用户输入: "明天下午3点开会讨论Q2的AI芯片选型进度"

    │
    ▼
┌─────────────────────────────────────────┐
│ 1. 前端发送 POST /task                  │
│    body: { text: "明天下午3点..." }      │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│ 2. TaskAgent 处理                        │
│                                         │
│    prompt: """                           │
│    解析以下任务文本，提取：                │
│    - title: 任务标题（简洁）               │
│    - date: 日期（YYYY-MM-DD）             │
│    - time: 时间（HH:MM，可选）             │
│    - priority: 优先级（high/medium/low）  │
│    - tags: 相关标签列表                    │

│    文本: "明天下午3点开会讨论Q2..."         │
│    """                                   │
│                                         │
│    output (结构化):                       │
│    {                                      │
│      title: "讨论Q2 AI芯片选型进度",       │
│      date: "2025-06-26",                 │
│      time: "15:00",                      │
```

### 2.3 标签系统的精确性设计

这是整个系统最关键的部分——标签必须**精确**而不是宽泛。

```
❌ 不好的标签（太宽泛/无用）
──────────────────────
#AI          #太宽泛，等于没标
#技术        #同样太宽泛
#笔记        #是笔记当然叫笔记
#随便记的    #无法用于检索
#今日思考    #没有信息量

✅ 好的标签（精确、可检索、可组合）
──────────────────────
#DeepSeek-R1                   #具体模型名
#推理性能测试                   #具体活动
#Qwen2.5-72B-对比              #对比对象
#长城AI科技战略-大模型选型        #战略方向
#R1-循环推理-问题               #具体问题
#AI芯片-昇腾-性能测试            #硬件+活动
#智能体-工具调用-架构设计         #技术领域+主题

标签命名规范:
  1. 具体实体直接用原名: #DeepSeek-R1, #Qwen2.5-72B
  2. 活动/事件用 动词+名词: #推理性能测试, #模型对比
  3. 战略方向用前缀: #长城AI科技战略-大模型选型
  4. 问题记录加后缀: #R1-循环推理-问题
  5. 多级标签用连字符: #AI芯片-昇腾-性能测试
```

```json
  "id": "task_20250625_001",
```

### 3.2 笔记（Note）— Markdown 文件

```markdown
---
title: DeepSeek-R1 推理速度测试
date: 2025-06-25
tags:
  - DeepSeek-R1
  - 推理性能测试
  - 模型对比
  - Qwen2.5-72B
  - 长城AI科技战略-大模型选型
type: test-report
status: draft
links:
  - [[Qwen2.5-72B-初步评测]]
  - [[大模型选型标准]]
---

今天测试了 DeepSeek-R1 的推理速度，结果比预期快约 30%。

## 测试结果
- 简单推理题: 平均 1.2s
- 复杂逻辑题: 平均 5.8s（偶尔出现循环推理）
- 代码生成: 平均 3.1s

## 对比
Qwen2.5-72B 在代码生成上稍好，但推理速度慢约 40%。
两者在不同任务上各有优劣。

## 下一步
准备撰写详细对比报告，包括成本、速度、质量三个维度。
```

**页面交互逻辑：**

```
① 输入任务 → POST /api/task → AI 解析时间 → 写入 tasks + schedule
                                  ↓
                          日历更新 + 任务列表更新
                          
① 输入笔记 → POST /api/note → AI 生成标签 → 保存 Markdown → 更新标签索引
                                  ↓
                          标签汇总区更新

② 点击日历日期 → GET /api/tasks?date=xxx → 显示该日任务详情

③ 点击任务 ✓ → PATCH /api/task/id/complete → 任务移动到 archive

④ 点击标签 → GET /api/notes?tag=xxx → 展开该标签下所有笔记列表

⑤ 点击笔记标题 → GET /api/note?id=xxx → 弹出笔记全文
```

## 七、关键技术决策

| 决策 | 选择 | 原因 |
|------|------|------|
| **Web 框架** | Flask | 简单、Python 原生、CLI 和 Web 可以共享 Agent 引擎 |
| **前端** | 原生 HTML + CSS + JS | 不需要前端构建工具，一个 HTML 文件搞定 |
| **日历组件** | 手写 JS 日历 | 不需要引入大库，月历渲染逻辑很标准 |
| **存储** | 文件系统（JSON + MD） | Claude Code 可直接读取，不需要数据库 |
| **AI 调用** | Claude API（agent.py 引擎） | 复用已有 Agent 引擎 |
| **标签精度** | AI 动态生成 + 精确描述 | 核心差异化能力 |
| **任务持久化** | 已完成任务永久归档到 JSON | 可查询历史 |
| **笔记持久化** | Markdown 文件到 vault | Claude Code 可直接 `grep`/`read_file` |

## 八、标签精确性 —— Prompt 设计要点

```python
NOTE_AGENT_SYSTEM = """你是一个围绕「长城AI科技战略」的知识管理助手。

## 标签生成规则（重要）
- 每个标签必须精确描述笔记的**具体内容**
- 禁止使用宽泛标签（如 #AI、#技术、#笔记、#思考）
- 使用具体术语（如 #DeepSeek-R1、#昇腾-推理性能、#智能体-工具调用）
- 多级标签用连字符连接（如 #AI芯片-昇腾-性能测试）

输出格式：Markdown 笔记（含 frontmatter）+ 精确标签列表
```
